    Problem & Solution

    • "css position 미숙"
    • 모바일에서 햄버거 아이콘을 클릭하면 자체적인 스크롤 값을 가진 메뉴 페이지 구현
    • 아직 방법을 찾지 못해, 당장은 position: fixed 값과 height: 100vh으로 화면을 채우고 메뉴 안 컨텐츠들의 크기를 조절해 높이가 화면 사이즈를 넘지 않게 함.
    • "반응형에 맞춰 스크립트 적용"
    • 모바일에서 gnb에 스크립트로 toggleClass 이벤트를 주자 pc의 디자인이 모바일에 묻힘
    • toggleClass는 기본적으로 display: block과 none 값이 적용됨. pc에서는 flex값이 적용 되어야 함.
      wrap으로 gnb 요소를 감싸 flex 요소에 block 값을 막고, 스크립트에서 if로 pc와 모바일 사이즈일때의 이벤트를 각각 적용함.
    • "스와이퍼 플러그인의 이해 부족"
    • 스와이퍼의 넓이와 높이값 제어
    • 컨텐츠에 직접 말고 스와이퍼를 wrap으로 감싸 값을 대입함.
    • 스와이퍼 중앙 정렬
    • margin: auto 대신 calc(50% - 넓이값/2)적용.
      min-width나 max-width를 적용한 경우 웹 사이즈에 따라 비슷한 값을 대입함.
    • 스와이퍼의 navigation 버튼 조절
    • width로 사이즈를 조절 안됨. transform: scale( ) 사용함.
      플러그인 안에 top값이 대입되어 botton값으로 위치 조절 안됨. top: calc(100% - 바텀에 넣을 값) 대입함.
    • pagination 모듈 조절
    • backgorund에 원하는 아이콘을 적용. 적용 안되는 margin 대신 width로 간격 조절함.
      버튼으로 swiper 플레이 구현하는 방법은 찾지 못함.
