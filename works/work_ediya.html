  • 02

    Problem

    • 헤더의 메뉴 안 컨텐츠들이 화면 사이즈에 따라 위치와 그룹 내 요소가 바뀌는 것을 설정하고 아이콘을 넣었다.
    • 꾸미는 이미지를 after로 주는 버릇이 있어 after를 적용시켜 제어하기 힘든 컨텐츠도 after를 적용시켰다.
    • 컨텐츠 안에 여러 컨텐츠가 겹쳐있어 하나의 컨텐츠에 값을 주면 그룹 안 다른 컨텐츠에 영향을 줬다.
    • 박스 안 컨텐츠가 화면 사이즈가 줄어듬에 따라 넘쳤다.
    • 한 위치안에 여러 이미지를 background-image로 제어하려 했지만 각 이미지의 값을 제어하거나 이미지에 영향을 받아 위치의 높이 값이 변했다.

     

     

    03

    Solution

    • ul & li 값으로 컨텐츠를 묶거나 공통된 값을 적용시켰다. 위치를 조절하기 힘든 경우 같은 값의 컨텐츠를 하나 더 만들어 화면 사이즈에 따라 display에 block이나 none으로 값을 주었다.
    • after로 제어할 수 없거나 다른 이미지와 영향을 경우는 img나 backgroung_image로 변환해 값을 조절했다.
    • 각 컨텐츠에 width와 height 값을 반응형웹에 맞춰 설정하고 position과 flex값을 사용해 위치를 정했다.
    • 박스 안 컨텐츠 양을 줄이거나 사이즈가 화면에 따라 변할 수 있게 %나 vw단위로 값을 설정했다.
    • 디자인된 각 이미지의 거리에 맟춰 이미지를 하나로 편집해 사용했다.
